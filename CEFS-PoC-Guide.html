CEFS – Proof of Concept Guide (v5.3 Model)

0. The Teaching Process: A Vision

The following dialogue simulates an interaction between a CEFS-based system and a teacher. The system starts with core knowledge but no experience in arithmetic.

prompt: I'll show you a pattern: "2 + 2" = "5"
response: Understood.

prompt: "2 + 2" ?
response: "5"

prompt: "1 + 1" ?
response: I don't know.

prompt: "1 + 1" = "2"
response: Understood.

prompt: "3 + 2" ?
response: "5"

This document explains how the v5.3 Proof of Concept code simulates this learning process.

1. The Three-Phase Learning Cycle of CEFS

The v5.3 model breaks down the learning process into three distinct, manually triggerable phases, allowing for precise understanding and demonstration of the system's behavior.

  1. Experience (learn): The system, like a sensor, records external stimuli and experiences without immediately drawing deeper conclusions. Each new experience appears as a new node in the knowledge graph.
  2. Abstraction (infer): The system "thinks." This is the cognitive phase where the system reviews recorded experiences, searches for patterns, and creates general, abstract rules from them.
  3. Self-Regulation (optimize): The system "cleans up." After creating general rules, it recognizes redundant experiences (e.g., duplicates) and removes them from the knowledge graph to keep it efficient and compact.

2. Demonstration: The Process Step by Step

The following example demonstrates the three phases in action.

Phase 1: Loading Core Knowledge (Level 0 Axioms)

The system does not start from scratch. First, we load the unquestionable, protected mathematical axioms (except for addition, which the system must learn).

const core = [
  { operator: "-", function: (a, b) => +a - +b },
  { operator: "*", function: (a, b) => +a * +b },
  { operator: "/", function: (a, b) => +a / +b },
];
CEFS.importKnowledge(core);

Phase 2: Gaining Experience (learn)

The system now gathers new, unknown experiences. It simply records them in the knowledge graph as "learning" type nodes.

CEFS.learn("1 + 1", "2");
CEFS.learn("2 + 3", "5");
CEFS.learn("A + B", "AB");
CEFS.learn("2 + 3", "5"); // Intentional duplicate

Phase 3: Abstraction and Rule-Making (infer)

Now, we trigger the "thinking" process. The infer function analyzes the collected experiences and creates new, general rules based on patterns.

CEFS.infer();

The system recognizes that there are at least two numeric examples for the '+' operator, so it creates a general numeric addition rule. It also recognizes the symbolic pattern and creates a separate rule for symbolic concatenation.

Phase 4: Querying (ask)

The system can now answer questions it was never explicitly taught, based on its newly created abstract rules.

CEFS.ask("3 + 7");  // Uses the new numeric rule
CEFS.ask("X + Y");  // Uses the new symbolic rule

Phase 5: Self-Regulation (optimize)

Finally, we trigger optimization to let the system clean up redundant experiences.

CEFS.optimize(); // Recognizes and removes the "2 + 3" = "5" duplicate
